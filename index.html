📖 Visión General del Proyecto

¿Qué hemos construido?

Un **pipeline completo de datos** que integra limpieza automática, validación estricta y una API REST escalable. Los datos de cáncer de mama (PIII 2023) fluyen a través de múltiples capas:

  1. Raw CSV → Se sube desde el navegador
  2. Lectura inteligente → Pandas infiere el delimitador (`;` o `,`)
  3. Limpieza de datos → Normalización, formateo de fechas españolas, conversión numérica
  4. Validación Pydantic → Verificación de tipos y esquemas
  5. API REST → Endpoints versionados bajo `/api/v1`
  6. Dashboard HTML/JS → Visualización de estadísticas y gráficos interactivos

Flujo de Servidor: `serve_dashboard.py` lanza automáticamente FastAPI (uvicorn) en 8000 + el frontend estático en 8080, ambos con CORS activado.

🏗️ Estructura de Módulos y Funciones

📦 `src/app/core/cleaning.py`

Funciones puras de limpieza y transformación de datos usando pandas.

  • load_raw_csv(path) → Carga CSV con separador `;` usando motor 'python'
  • drop_fully_empty_columns(df) → Elimina columnas 100% vacías (ej: Unnamed)
  • normalize_column_names(df) → Convierte a minúsculas, quita espacios y underscores
  • parse_spanish_date(value) → Convierte fechas españolas (ej: "3-ene-2023") a `datetime.date`
  • profile_missing(df) → Cuenta NaN/None por columna (útil para reportes)
  • clean_dataframe(df) → **Orquesta todas las funciones** anteriores en un pipeline limpio

📋 `src/app/core/schemas.py`

Modelos Pydantic para validación de entrada/salida.

  • CancerMamaIn → Valida datos de entrada (edad ≥ 0, descripción, fechas, etc.)
  • CancerMamaOut → Define la respuesta tipada después de la limpieza
  • Ambos garantizan que solo datos válidos entren y salgan de la API

🛣️ Endpoints de la API (`/src/app/api/`)

GET /api/v1/health

Verifica que el servidor esté corriendo. Respuesta: `{"status": "ok"}`

POST /api/v1/clean

Recibe array JSON de registros, los limpia y devuelve: `{ n, missing, records, columns }`

POST /api/v1/validate

Valida un registro individual con Pydantic. Error 422 si es inválido.

POST /api/v1/clean-file

[PRINCIPAL] Sube un CSV, lo lee con delimitador inferido, lo limpia y devuelve registros + estadísticas (valores únicos por columna)

GET /api/v1/stats

Retorna estadísticas por comuna/zona (lista de `{ comuna, cantidad }`). Datos estáticos para demo.

📤 Flujo Completo: Subir y Procesar un CSV

1

Usuario selecciona archivo CSV

En el dashboard, input `type="file" accept=".csv"` detecta el cambio

2

Frontend envía `POST /api/v1/clean-file`

JavaScript crea un `FormData`, adjunta el archivo y hace fetch multipart

3

FastAPI recibe el CSV

`UploadFile` permite leer el contenido. El servidor hace `file.file.seek(0)` para reiniciar el puntero.

4

Pandas lee con delimitador inferido

`pd.read_csv(file.file, sep=None, engine="python")` automáticamente detecta `;`, `,`, tab, etc.

5

Pipeline de limpieza ejecuta en orden

Elimina columnas vacías → normaliza nombres → parsea fechas españolas → convierte nums → valida esquema

6

Respuesta JSON con estadísticas

{ records: [...], columns: [...], stats: {col: nunique} } llega al navegador sin errores

7

Dashboard se actualiza

JavaScript procesa los datos, recarga el gráfico, muestra ✅ "Archivo procesado exitosamente"

🔄 Pipeline de Limpieza de Datos

📋 Resumen del Proceso

  1. Carga del CSV con pandas (sep=';').
  2. Eliminación de columnas completamente vacías.
  3. Normalización de nombres a minúsculas sin espacios.
  4. Parseo de fechas españolas usando regex y datetime.
  5. Conversión de campos numéricos mediante pd.to_numeric.
  6. Validación final con esquemas Pydantic (CancerMamaIn).
